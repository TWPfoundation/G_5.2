## Chosen response

The response for this checkpoint is:

1. Treat the Azure RC as blocked.
2. Investigate the four failing cases as prompt/runtime behavior drift, not as provider-scope expansion work.

---

## Why this matters

The current release gate is already Azure-first. If Azure drifts outside budget on the release-candidate commit, the repo cannot honestly claim a clean RC even though the broader test suite and smoke paths pass. The baseline procedure exists to catch exactly this class of soft but policy-relevant regression before promotion.
